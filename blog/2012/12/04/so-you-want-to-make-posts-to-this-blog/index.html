So You Want to Make Posts to This Blog

Dec 4th, 2012 | Comments

I decided to use Octopress for this blog since it works with Github Pages , it’s got HAML support and it’s a self-described blogging framework for hackers. As it turns out, the procedure for actually making a post is pretty technical. I don’t want to the be the only one making posts about our Hack-A-Thon(s), so here is the procedure for making posts.

Octopress is a self-described blogging framework for hackers. Most of the folks in the BOSS meetup fit that description, so it shouldn’t be too tough for anyone. You’ll need a Github account, some rudimentary ruby and git skillz, and a burning desire to put something on the Interwebs. I’ll use HAML for most of my posts, but you can use Jekyll or Markdown if you want.

Let’s get started!

  • Fork it!

    Make a fork of git@github.com:bosshack/bosshack.github.com.git

  • Clone your repo and go to that directory (probably bosshack.github.com )

    git clone git@github.com:YOURGITHUBIDHERE/bosshack.github.com.git
    Most people will need to make changes to their own fork instead of the master

  • Add a remote named upstream that points to the master repo

    git remote add upstream git@github.com:bosshack/bosshack.github.com.git
    You may want to send a pull request if you have a post or changes to a post

  • Delete the contents of the _deploy directory if it exists. It shouldn’t

    rm -rf _deploy
    We’re gonna clone your fork again into the _deploy directory

  • Checkout the source branch

    git checkout source
    The source branch is where the posts and pages are written in your markup style of choice.

  • Clone your fork to the _deploy directory (seems redundant, right? - it’s kind of cool actually)

    git clone git@github.com:YOURGITHUBIDHERE/bosshack.github.com.git _deploy

  • Bundle that Sh*t

    bundle

  • Preview the site on port localhost:4000

    rake preview
    This command will watch the source directory for changes and regenerate the site when something changes

  • Make a post

    rake new_post["My Awesome Post Title"]
    This creates a new file in the source/_posts directory with a filename like YYYY-MM-DD-my-awesome-post-title.haml with YAML Front Matter. Feel free to change the extension to .markdown if you would rather edit in Markdown.

  • Edit the file above as appropriate. Continually check your work in the browser and in the session where you ran
    rake preview
    Any HAML/Markdown/etc errors will show up in the STDOUT of that command.
  • Add your post to the git source branch and commit

    git add source/_posts/*
    git commit -m "Created an awesome blog post for bosshack.org"

  • Push the changes to your fork and send a pull request to bosshack if you want it included on the site.
  • If you run into any problems with these instructions, leave a comment below and I’ll update them accordingly
